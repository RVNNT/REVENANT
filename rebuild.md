# Стратегія розгортання документації: Онлайн та Офлайн інфраструктура

Для забезпечення максимальної автономності, простоти та універсальності, найкращим вибором є комбінація **Markdown** (як єдиного джерела правди) та інструменту **Pandoc** або мінімалістичного **Static Site Generator (SSG)**.

Нижче наведено три варіанти реалізації — від абсолютно незалежного від стороннього софту до найбільш автоматизованого.

---

## Варіант 1. Скрипт на Bash + Pandoc (Максимальна автономність)

**Pandoc** — це "швейцарський ніж" для конвертації текстових файлів. Він скомпільований під усі можливі ОС (включаючи застарілі архітектури) і не вимагає середовища виконання типу Node.js чи Python.

* **Як це працює:** Ви пишете документацію у форматі Markdown. Скрипт автоматично збирає всі `.md` файли, додає до них однаковий HTML-шаблон з навігацією та генерує статичний сайт.
* **Чому це підходить:** Якщо інфраструктура "впаде", у вас залишається Bash-скрипт на 20 рядків, який можна запустити та відредагувати на будь-якому комп'ютері.

### Приклад архітектури збірки:

```bash
#!/bin/bash
# Простий цикл для генерації сторінок

for file in volume_*/*.md; do
    html_file="${file%.md}.html"
    # Pandoc об'єднує заголовок, навігацію та контент Markdown в один HTML
    pandoc "$file" -o "$html_file" --template=templates/layout.html
done
